# リズムクルーズ デザインシステム

## 1. 目的

この資料は、リズムクルーズの現在のデザインを、今後の Sound Cruise 系新規アプリで踏襲しやすくするための参照資料です。

対象は、デザイン原則、画面構造、UI パターン、ボタン、カード、配色、戻る導線、PRO / 通常版の見せ方です。アプリ本体の仕様書ではなく、内部ロジックや保存構造の説明資料でもありません。

この資料は HTML / CSS / JS を共通化する目的ではありません。既存 class をそのまま新規アプリへコピーすることを推奨するものではなく、見た目や構造の考え方を参照するための資料です。

マイク補正、判定、保存、localStorage、service worker、PRO 認証の実装詳細は対象外です。

### この資料の使い方

新規アプリ制作時は、まずこの資料を参照します。使う順番は以下を目安にします。

1. デザイン原則を読む。
2. TOP / 戻る導線 / 設定画面の構造を確認する。
3. ボタン / カード / 配色の考え方を参照する。
4. 新規アプリ側の用途に合わせて、class名やDOMは作り直す。

この資料は、リズムクルーズの HTML / CSS / JS をそのままコピーして共通化するためのものではありません。「流用しやすい」はコピー推奨ではなく、UI パターンを参考にするという意味です。

## 2. 基準バージョン

- 対象アプリ: `apps/rhythm-cruise/`
- 基準バージョン: `v0.12.101`
- 基準 commit: `8169a43`
- commit message: `通常マイクの表示名をイヤホンなしに変更`
- 作成時点の未追跡アイコン案 `rhythm-cruise-icon-proposal.png` と `rhythm-cruise-icon-proposal.svg` は、この資料の対象外です。

この資料に出てくる class名、ID、DOM構造は、基準バージョン `v0.12.101 / commit 8169a43` 時点の実装をもとにしています。将来のリズムクルーズでは class名やDOM構造が変わる可能性があるため、新規アプリで参照するときは、その時点の実装や最新資料も確認します。この注意は、アプリ本体の class名やDOMを変更する意味ではありません。

## 3. デザイン原則

- 黒から焦げ茶の暗い背景をベースにする。
- アンバー / ゴールドを主アクセントにし、Sound Cruise 系らしい温度感を出す。
- スマホ縦画面を優先し、最大幅はおおむね `600px` に収める。
- TOP では迷わせず、最初の行動を大きなカード型導線で示す。
- 大きなカード型導線、STAGE カード、設定カードを中心に画面を組む。
- スマホ縦画面で読みやすい文字サイズを優先し、見出し、本文、補足文の情報階層を明確にする。
- タップ要素は指で押しやすい高さを確保する。厳密な固定値ではなく、主要ボタンやカード導線は44px前後以上のタップ領域を意識する。
- 小さな補助ボタンは主導線より控えめにするが、押せないほど小さくしない。
- 設定は情報を小分けにし、1カード1目的に近づける。
- 長い説明や開発情報は `details` などで折りたたみ、初見の視線を邪魔しない。
- 主導線とサブ導線を見た目で明確に分ける。
- 危険操作は画面下部や専用セクションへ隔離し、通常操作と混ぜない。
- PRO 要素は金色、バッジ、ロック表示で表現する。
- 開発情報、詳細ログ、診断値は奥に畳み、通常利用では目立たせない。

## 4. TOPページ構造

### 通常版

- TOP の親は `#screen-home`。
- タイトルは `header > h1` に置く。
- 最初に表示される導線群は `#home-top`。
- メイン導線は `.rc-category-card` を使い、アイコン、主ラベル、補足文の3段構成にする。
- 右上には `.settings-corner-btn` を固定表示する。
- 左上の共通ナビは `#app-nav.app-nav`。TOP では隠し、サブビューや Practice / 設定画面で表示する。
- 下部には `Sound Cruise Series` を控えめに表示する。

### PRO版

- PRO版は `html data-app-edition="Pro"` で版を表す。
- TOP タイトルは `.pro-home-title` を使い、タイトル文字と `.pro-badge` を横並びにする。
- 通常版と PRO版の画面構造は近く保ち、差分は PRO 表示、PRO 認証、利用可能機能に寄せる。
- PRO版だけ `pro-gate.css` / `pro-gate.js` を読み込む。
- PROゲート読み込みはデザイン資料では触れるだけにし、実装や認証仕様は変更しない。

## 5. 戻る導線・ページ構造

- 共通ナビは `#app-nav.app-nav`。
- 戻るボタンは `#nav-back-btn`、文言は `← 戻る`。
- TOP ボタンは `#nav-top-btn`、文言は `🏠 TOP`。
- TOP では共通ナビを隠す。
- `TOP` は常にホームTOPへ戻す。
- `戻る` は現在位置から自然に一階層戻る。
- ホーム内サブビューは、TOP、リズム練、リズム作成、編集、保存済み一覧のような階層として扱う。
- Practice 画面から戻るときは、開始元に応じて STAGE 一覧、リズム作成、編集画面などへ戻る。
- 作業中、補正中、テスト中、未保存の可能性がある場合は、中断確認を挟む。

新規アプリで踏襲するルールは、`TOP は絶対復帰`、`戻るは一階層戻る`、`作業中は中断確認` の3点です。

## 6. ボタンデザイン

| class | 用途 | 見た目の特徴 | 流用しやすさ | 注意点 |
|---|---|---|---|---|
| `.btn-primary` | 開始、適用、再生などの主操作 | オレンジ系グラデーション、太字、丸み強め、影あり | 高い | 画面内で主役を増やしすぎない |
| `.btn-ghost` | 補助操作 | 透明背景、薄い白枠、控えめな文字色 | 高い | 主操作と同じ重みにしない |
| `.btn-mini` | 小さな補助操作 | 透明背景、低コントラスト、小さめ | 高い | 重要操作には使わない |
| `.btn-mini.btn-reset` | リセット、戻す操作 | 薄い枠付き、押せることが分かる | 中 | 初期化や削除と混同しない |
| `.settings-top-main` | 設定TOPの主導線 | オレンジ枠、淡いオレンジ背景、横幅いっぱい | 高い | 設定カード内の一番大事な操作に限定する |
| `.settings-top-preset` | 保存済みプリセット導線 | 淡いオレンジ、主導線より控えめ | 高い | 主導線ではなく準主導線として扱う |
| `.settings-top-sub` | 現在設定を見る、手動設定など | 白枠、淡い背景 | 高い | 設定の補助導線に使う |
| `.nav-btn` | 戻る / TOP | 左上固定、ピル型、暗色背景、アンバー枠 | 高い | TOPでは隠す |
| `.settings-corner-btn` | 右上設定 | 右上固定、ピル型、歯車付き | 高い | 設定画面では隠す |
| `.rc-category-card` | TOPの大きな導線 | 大きなカード、アイコン、金色ラベル | 高い | class名のコピーより構造の考え方を参照する |

危険操作ボタンは、通常の主導線から離し、画面下部や `危険操作` セクションへ置きます。赤系の枠や文字色を使い、保存、開始、戻ると混ざらないようにします。

## 8. 配色・余白・角丸・影

主要変数:

```css
--bg-color: #120f0c;
--surface-color: #1d1813;
--surface-raised: #251f18;
--primary-color: #ff9f1c;
--secondary-color: #ffd166;
--accent-cool: #4d96ff;
--text-color: #fdf6ee;
--muted-color: rgba(253, 246, 238, 0.62);
--error-color: #ff4f6e;
--early-color: #4d96ff;
--just-color: #2ecc71;
--late-color: #ff6b6b;
```

- 背景は `--bg-color` をベースに、上部から淡いアンバーの radial-gradient を重ねる。
- カード背景は `--surface-raised` から `--surface-color` への縦グラデーションが基本。
- ボタン背景は主操作がオレンジ系、サブ操作が透明または白の薄い背景。
- 角丸はカードが `16px` から `22px`、設定カードが `18px`、小ボタンが `10px` から `14px`、ピル型が `999px` または `50px`。
- 影は黒の大きめシャドウと、内側の薄い白ハイライトを組み合わせる。
- 境界線は白の低透明度、またはアンバーの低透明度を使う。
- 余白は外側 `20px`、カード内 `20px` から `24px`、カード間 `12px` から `18px` が中心。
- 注意色 / 危険色は赤系、成功色は `--just-color`、タイミング判定は `--early-color` / `--just-color` / `--late-color` で表す。

## 9. PRO / 通常版の見せ方

- PRO版は `html data-app-edition="Pro"` で表す。
- 通常版と PRO版の画面構造は近く保つ。
- PRO版だけ `pro-gate.css` / `pro-gate.js` を読み込む。
- PRO版のタイトルには `.pro-home-title` と `.pro-badge` を使う。
- PRO要素は金色アクセント、PROバッジ、ロック表示で見せる。
- 通常版でも PRO 要素を見せる場合は、押せる導線にロック案内を付け、何がPRO限定か分かるようにする。
- 通常版で完全に隠すか、見せてロックするかは、機能の訴求価値と誤操作リスクで判断する。
- 新規アプリでは PRO 認証そのものは `apps/shared` 側に任せ、この資料では見せ方だけを扱う。

## 10. 設定画面構造

- 見出しは `.settings-head`。
- 大分類は `.settings-tabs` と `.settings-tab` で切り替える。
- 設定内容は `.settings-card` に小分けする。
- 設定TOPでは、主導線を `.settings-top-main`、サブ導線を `.settings-top-preset` / `.settings-top-sub` で分ける。
- 説明は `.card-help` に畳む。
- 長い説明、詳細値、開発ログは初期表示に置かない。
- 危険操作や初期化ボタンのような操作は、通常操作から離して下部に置く。
- マイク補正の詳細ロジックは流用対象外です。カード分割、主導線とサブ導線の整理、説明折りたたみの考え方だけを参照します。

- class名を変更しない。
