# リズムクルーズ デザイン参照スクリーンショット

## 1. 目的

このフォルダは、リズムクルーズのデザインを今後の新規アプリへ踏襲するときに参照するためのスクリーンショット集です。

実装資料ではなく、見た目、画面構造、導線、カードUI、ボタン配置の雰囲気を確認するための資料です。HTML / CSS / JS を共通化する目的ではありません。

## 2. 紐づく資料

- 紐づく資料: `../DESIGN_SYSTEM.md`
- 基準バージョン: `v0.12.101`
- `DESIGN_SYSTEM.md` 追記済み commit: `485700fa49952cf5c07851b3d21555730b2a00ea`

このスクリーンショット集は、上記の基準バージョンと資料に紐づく見た目の参照です。将来のリズムクルーズで画面やclass名、DOM構造が変わる可能性があるため、新規アプリ制作時は最新の資料と実装も確認します。

## 4. 撮影できなかった画面

### 結果カード

- 画面名: `05_results_card.png`
- 理由: 実際の結果カードを表示するには、Practice画面で練習を進めて結果状態を発生させる必要があります。今回はマイク許可や実機依存の操作、結果表示のためだけの無理なテスト実行を避けました。
- 今後撮るなら: 通常の動作確認中に結果カードが自然に表示されたタイミングで、同じスマホ縦画面相当のサイズで撮影します。

### PRO版 TOP

- 画面名: `11_pro_top.png`
- 理由: PRO版ページではPRO認証ゲートが表示されます。今回はPRO認証状態、localStorage、sessionStorage、アプリ内部状態を変更しない方針のため、TOP画面まで進めず未撮影としました。
- 今後撮るなら: 正規の認証済み状態でPRO版を開ける確認環境を使い、PRO認証や保存構造を変更せずに撮影します。

## 5. 注意点

- スクショは見た目の参照用であり、コード流用の根拠ではありません。
- class名やDOM構造は `../DESIGN_SYSTEM.md` 側を参照します。
- アプリ内部ロジック、補正、判定、保存、PRO認証は対象外です。
- 新規アプリでは、この見た目を参考にしつつ用途に合わせて設計し直します。
- スクショに写っているボタンやカードは、HTML / CSS / JS を共通化するためのものではなく、UIパターンを参考にするためのものです。
