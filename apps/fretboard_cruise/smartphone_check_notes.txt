キャッシュ対策内容
- Standard / Pro の index.html で script.js の読み込みクエリを 1.138.3 に更新。
- Standard / Pro の index.html で theme.css の読み込みクエリを 347 に更新。
- Standard / Pro の index.html で manifest.json の読み込みクエリを 1.138.3 に更新。
- theme.css の中身は変更していない。
- Standard の service worker キャッシュ名を fretboard-cruise-standard-v1.138.3 に更新。
- Pro の service worker キャッシュ名を fretboard-cruise-pro-v1.138.3 に更新。
- service worker は install 時に skipWaiting、activate 時に既存キャッシュ削除と clients.claim を行うため、更新バーまたはページ更新で最新版へ切り替わる想定。

PWAアイコン
- Standard / Pro とも同じ最終選定アイコンを使用。
- apple-touch-icon: generated-home-icons/final-selected-icon/fretboard-cruise-final-selection-180.png
- manifest 192px: generated-home-icons/final-selected-icon/fretboard-cruise-final-selection-192.png
- manifest 512px: generated-home-icons/final-selected-icon/fretboard-cruise-final-selection-512.png
- manifest では any と maskable を分けて登録。
- 画像は正方形のまま書き出し、縦横比は変更していない。

Pro版の維持事項
- data-app-edition="Pro" は維持。
- pro-gate.css / pro-gate.js の読み込みは維持。
- window.__SOUNDCRUISE_PRO_GATE__ の password は変更なし。
- GATE_VERSION は変更なし。
- manifest.json の start_url と scope は "./" のまま維持し、新しいProディレクトリ配下で完結。
- service worker は新しいProディレクトリ配下の service-worker.js として登録される。

スマホ確認時の注意点
- iPhone / Android とも、まず上記URLをブラウザで直接開く。
- 既にホーム画面追加済みのPWAがある場合は、起動後に更新バーが出たら「ページを更新」を押す。
- 更新バーが出ない場合でも、ブラウザで再読み込みして Ver. 1.138.3 表示になっているか確認する。
- 古い表示が残る場合は、該当PWAを一度終了して再起動する。
- それでも切り替わらない場合は、旧ホーム画面アイコンを削除し、新しいURLから再追加する。

PWA更新時の注意点
- Standard と Pro は別ディレクトリ、別 manifest、別 service worker scope で動作する。
- Pro のURLディレクトリが変わったため、旧Pro URLでホーム画面追加済みのPWAは新URLへ自動移行されない。
- Pro版をホーム画面で確認する場合は、新しいPro URLから追加し直す。
- 旧Pro URLの互換維持は行っていない。
