# クルーズスタジオ — アプリコンセプト

## 1. 目的

クルーズスタジオは、**一人で音楽制作・アレンジを完結させるための母艦アプリ**です。

最初のターゲットは弾き語りスト。「弾き語り教材用のコード譜を作る」ところから始めて、
同じ曲データを伴奏MIDI生成（プレDTM）や練習アプリ連携へ広げていきます。

- 対象ユーザー: 弾き語りスト、教材を作るギター講師、一人で曲を仕上げたい制作者
- 対象環境: **PCファースト**（譜面編集・歌詞入力・PDF出力・MIDI書き出しが中心のため）
- 配信形態: Sound Cruise シリーズと同じ静的ホスティング（GitHub Pages / soundcruise.jp）

## 2. 母艦アプリとしての位置づけ

クルーズスタジオは「アプリ集」ではなく、**1つの曲データ（StudioProject）を中心に
複数のモジュールが取り囲む構造**です。

```
                ┌─────────────────────────┐
                │      StudioProject      │  ← 共通曲データ（DATA_MODEL.md）
                │  コード / 歌詞 / メロディ │
                │  キー / カポ / BPM / 拍子 │
                └───────────┬─────────────┘
        ┌───────────┬───────┴───────┬──────────────┐
   譜面クルーズ  プレDTMクルーズ   将来連携（リンク起動）
   （表示・編集） （MIDI変換・試聴）  音感 / 指板 / リズム
```

## 3. 各モジュールの役割

### 譜面クルーズ（最初に完成させる）

弾き語り教材用のコード譜エディタ。1枚のA4に以下をまとめる。

- 曲名 / アーティスト / ORGキー / Playキー / カポ / BPM
- 基本ストロークパターン
- 小節ごとのコード進行
- 歌詞（音節単位）
- ドレミ（メロディ階名、オクターブ表示付き）
- セクション（イントロ / Aメロ / サビ …）
- ブラウザ印刷ベースのPDF出力

### プレDTMクルーズ（将来）

譜面クルーズと**同じ StudioProject** から伴奏MIDIを生成する。

- アコギストロークMIDI（コード × ストロークパターン）
- ベースのルート音8分MIDI（コードのルートから）
- ドラム8ビートMIDI（BPM・拍子から）
- ブラウザ試聴 / SMF(.mid)書き出し / Logic・Cubase・Studio One への橋渡し

### 既存アプリ連携（将来）

音感クルーズ・指板クルーズ・リズムクルーズは**統合せず、リンク起動＋URLパラメータ連携**を基本とする。
既存アプリ側には変更を加えない。詳細は docs/DECISIONS.md 参照。

## 4. 中心思想

**「譜面は曲データのビュー、MIDIは曲データの変換」**

- 曲データ（StudioProject）が唯一の正。見た目のためのデータを持たない
- 譜面クルーズ = StudioProject を人が読みやすくレンダリングしたもの
- プレDTMクルーズ = StudioProject をMIDIイベントにコンパイルしたもの
- したがって、すべてのイベント（コード・歌詞・メロディ・ストローク）は
  **小節番号＋小節内tick（ticksPerBeat=480）** に紐づける
- メロディは絶対音名ではなく**階名（movable do の度数）＋オクターブ**で持ち、
  キー変更・カポ変更に対して壊れないようにする

この思想を破る変更（例: 譜面専用の座標データを曲データへ入れる）は、
docs/DECISIONS.md に記録してから行うこと。

## 5. MVPでやること / やらないこと

### やること（Phase 1〜3）

- 母艦TOP（モジュールカード一覧）
- StudioProject スキーマと生成・検証・マイグレーション
- localStorage 自動保存 + JSONファイルの書き出し / 読み込み
- 譜面クルーズ: 曲情報・セクション・小節グリッド・コード・歌詞・ドレミの編集
- A4固定レイアウトの譜面プレビューとブラウザ印刷

### やらないこと（将来フェーズ or 対象外）

- 五線譜 / TAB の本格描画（Phase 6。VexFlow vendor を想定した拡張余地のみ残す）
- MusicXML 互換（Phase 6。まず独自JSONが本命）
- MIDI生成・試聴（Phase 5）
- PRO認証（Phase 8。shared/pro-gate.js の共通思想に合流する）
- service-worker / manifest / PWA化（当面作らない。docs/DECISIONS.md 参照）
- 既存アプリの改修・統合・スマホ向け編集UIの最適化
- 既存楽曲の歌詞・メロディをサンプルデータとして持つこと（著作権上の理由）

## 6. 設計ドキュメントの構成

| `APP_CONCEPT.md` | 本書。目的・構造・中心思想・やらないこと |
